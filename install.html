Local smoke test

Open smoke.html and click Emit sample traffic to confirm the generated script can mount the overlay, hook a page WebSocket, parse a local round/draw/discard sample, update the live gate, and export a capture before using a real game page.

Before real-page sampling

  1. Open Mahjong Soul and confirm the overlay appears.
  2. Click Self-test in the debug panel and confirm it reports Self-test: ok.
  3. Confirm the debug panel shows Install: installed, client decode hooked when the game code has loaded, hook diagnostics, and a non-zero socket count after joining a table.
  4. Watch the MVP gate line. If it lists missing items, collect from the start of a round or follow the missing field names before trusting the state.
  5. Watch the Real-page preflight line. It should reach 15/15 before export, including the safety snapshot checks, then run the displayed capture-doctor command before the final real-page gate.
  6. Keep realtime advice off unless explicitly testing training-mode analysis; acceptance samples require liveSafetySettings to show realtime advice off, manual input clear, and automation/message mutation disabled.
  7. Use Download capture, then run npm run import-capture -- path/to/majsoul-helper-capture.json, or use Copy capture and save the JSON locally. If import prints Import notice, the file was copied but still lacks one or more real-page proof fields, so continue with the doctor/gate output before treating it as accepted. Then run npm run capture-doctor -- captures/capture-real.json, npm run replay -- captures/capture-real.json, and npm run real-page-gate.

Detailed sampling checks are in docs/real-page-sampling.md.
